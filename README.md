**WEB-OHJELMOINNIN SOVELLUSPROJEKTI**

_Projektin tarkoituksena oli tehdä toimiva ruoka-websovellus eli sovellus, jossa asiakkaalla on 
mahdollista tilata ruoka-annoksia ja ravintolan omistajalla on mahdollista lisätä ravintoloita ja ruoka-annoksia.  Projektimme on tehty käyttäen Java-, JavaScript- ja HTML-ohjelmointikieliä, sekä Heroku-pilvipalvelualustaa. Koko projektin toteutus löytyy GitHubista._

**MEISTÄ**

Ryhmä 5 koostuu henkilöistä Aliisa Kortesoja, Teemu Lahtinen, Mikko Turpeinen ja Casandra Andersson. Projektin alussa jaoimme kaikille seuraavat vastuualueet: 

Backend - Teemu

React - Mikko

Frontend ja UI-suunnittelu - Aliisa

Frontend ja viestintä - Casandra


Kokoontuessamme Teams-palvelussa tai koululla, kaikki opiskelijat pyrkivät osallistumaan jokaiseen osa-alueeseen, jotta projektissamme korostuisi toimiva ryhmätyö ja laajempi oppiminen projektin aihealueista. Aloitimme työskentelyn 14.3.2022 ja toimivaksi testattu sovellus on palautettu 18.4.2022. Kokonaisuudessaan projektiin meni viisi viikkoa ja olemme vähintään kerran viikossa käyneet läpi miten omat vastuualueen tehtävät ovat edenneet sekä pitäneet yhteisiä työpajoja. Suunnittelimme ja kokosimme web-sovelluksen, jossa rekisteröityneellä käyttäjällä on mahdollista tilata ruokaa haluamastaan ravintolasta sekä selata tilaushistoriaansa. Ravintoloitsijalla on mahdollista lisätä ruoka-annoksia ja uusia ravintoloita, sekä selata omien ravintoloidensa tilaushistoriaa.

_Kuva 3: Käyttäjän luonti_

		
**AVAINASIAT**
Projektimme alussa valitsimme ruokasovelluksemme alustaksi pilvipalvelualustan Herokun, koska se tuki niitä ohjelmointikieliä, joita sovelluksen luontiin oli tarkoituksena käyttää. Lisäksi Herokun integraatio GitHubiin puolsi valintaa, GitHubin ollessa tärkeä työkalu projektin kannalta. 
 Käyttöliittymän suunnittelussa hyödynsimme Figma -sovellusta, jolla on mahdollista suunnitella sovelluksen ulkoasu ja käyttöliittymä. Sovelluksen backendin päädyimme toteuttamaan Javalla, sen ollessa kaikille ryhmäläisille tuttu ohjelmointikieli. 
Tietokanta valintanamme oli Heroku Postgre SQL.
